Privacy Policy

Smartsheet Controller — Chrome extension · Last updated: April 28, 2026

Summary

This extension is a thin client: it opens your own Smartsheet Controller web app in the side panel next to Smartsheet. It does not send your Smartsheet API tokens, chat content, or spreadsheets to the extension author. You choose the server URL (for example your self-hosted FastAPI instance).

What the extension processes

  • Smartsheet tab URLs — To detect a numeric sheet ID from app.smartsheet.com paths such as /sheets/<id>. Only the ID and the fact that a sheet was opened are stored temporarily in the browser session (chrome.storage.session) for the side panel.
  • Controller server URL — The origin you save in extension settings (for example http://127.0.0.1:8100) is stored in sync storage (chrome.storage.sync) so your devices can share the same preference if you use Chrome sync.
  • Optional OAuth token — If you use “Sign in with Smartsheet” from the options page, the access token returned by your Controller server may be stored locally in the extension (chrome.storage.local) so you can copy it into the main app. It is not transmitted to the extension author.

    • Permissions (why we ask)

    • storage — Saves your Controller base URL and extension-related preferences.
    • sidePanel — Shows the Controller UI in Chrome’s side panel next to your tabs.
    • tabs — Reads the active tab’s URL when it finishes loading so we can extract a sheet ID from Smartsheet. We do not read page content, passwords, or form fields on Smartsheet.
    • identity — Used only for optional “Sign in with Smartsheet” OAuth from the options page: Chrome opens Smartsheet’s authorization page and returns the redirect URL to the extension. We do not access your Google account.
    • Host access (Smartsheet + your Controller) — Lets the side panel load your Controller in an iframe and inject a small stylesheet on that origin only to simplify the layout when embedded. Data flows between your browser and the server you configured, not to us.

    What we do not do

    • We do not sell or rent personal data.
    • We do not run analytics or ads inside the extension.
    • We do not operate the Smartsheet Controller backend for you — you or your organization do.

    Your Controller server

    If you point the extension at a server you control, that server’s privacy practices apply to tokens, chat history, and API calls. Review that project’s documentation.

    Changes
